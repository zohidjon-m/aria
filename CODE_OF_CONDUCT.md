# Code of Conduct

## Our Standard

This project welcomes respectful, technically grounded collaboration. AML,
banking, security, and AI governance topics can have high stakes, so discussions
should stay clear, evidence-based, and focused on the work.

Participants are expected to:

- be respectful and constructive,
- assume good faith while asking for evidence,
- avoid harassment, threats, insults, and discriminatory language,
- protect confidential, proprietary, regulated, and personal data,
- avoid posting real customer records, credentials, suspicious activity details,
  SAR-related confidential information, or bank-internal material.

## Unacceptable Behavior

Unacceptable behavior includes harassment, intimidation, doxxing, sexualized
language, hate speech, repeated disruption, public disclosure of sensitive
security issues, or sharing regulated/private data without authorization.

## Enforcement

Maintainers may remove comments, close issues, block accounts, or report abuse
to hosting providers when behavior violates this code. Security issues should be
reported through the private process in [SECURITY.md](SECURITY.md).
